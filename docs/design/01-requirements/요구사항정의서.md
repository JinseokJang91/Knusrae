## 1. 개요

### 1.1 목적

본 문서는 Knusrae 애플리케이션에 **구현된 기능**을 분석하여 도출한 기능·비기능 요구사항을 정의한다.  
요구사항은 실제 구현된 화면·API·도메인 기능을 기준으로 정리하였다.

### 1.2 적용 범위

- **대상**: Knusrae 웹 애플리케이션 (프론트엔드 Vue 3 + 백엔드 Spring Boot 마이크로서비스)
- **범위**: 회원·인증, 레시피, 검색·카테고리, 대시보드·추천, 팔로우·크리에이터, 북마크·찜, 댓글·조회, 재료·문의, 관리자, 공통(FAQ·에러) 등 전 영역

### 1.3 용어 정의

| 용어 | 설명 |
|------|------|
| 크리에이터 | 레시피를 등록·공유하는 회원 |
| 레시피북 | 사용자가 만든 북마크 폴더(모음) |
| 북마크 | 레시피를 레시피북에 저장한 단위 |
| 찜 | 회원이 레시피를 좋아요한 관계 |

| NFR-03 | 역할 기반 접근(관리자) | 관리자 전용 API·화면은 관리자 역할만 접근 | 비기능 | 필수 | 구현됨 | ADMIN 권한 |
| NFR-04 | API 캐싱 | 추천·인기·테마·카테고리·크리에이터 등 일부 API 캐시 적용 | 비기능 | 일반 | 구현됨 | Cache-Control, refresh 파라미터 |
| NFR-05 | 페이지네이션 | 목록 조회 시 limit/offset 기반 페이지네이션 지원 | 비기능 | 필수 | 구현됨 | API·UI 공통 |
| NFR-06 | 공통 에러 처리 | API 공통 에러 응답 형식·예외 핸들링 | 비기능 | 필수 | 구현됨 | GlobalExceptionHandler 등 |
| NFR-07 | CORS·보안 설정 | 외부 도메인·헤더 등 보안 설정 | 비기능 | 필수 | 구현됨 | SecurityConfig |
